# UI Guidelines

## 1. Design Philosophy
The InternReach / Studojo design is a **Modern Professional SaaS** aesthetic. It aims to evoke confidence and clarity during a high-stakes activity (job hunting).

### Core Tenants:
- **Card-Centric**: Information is always grouped into distinctive, high-radius white cards.
- **Calm Layouts**: Heavy use of whitespace and the `bg-page` neutral gray to prevent cognitive overload.
- **Modular Color Coding**: The color (Purple, Green, or Orange) tells the user which "Dojo" (module) they are currently in.

---

## 2. Content Constraints

### Emojis and Graphics
- **No Emojis**: Emojis are strictly forbidden in the primary UI text.
- **Iconography**: Use clean, line-weight icons (feather-style) consistent with professional tools.

### Copywriting Tone
- **Professional**: Instruction text should be concise and supportive.
- **Clarity**: Use active verbs for buttons (e.g., "Build Resume" instead of "Submit").

---

## 3. Presentation Standards

### Component Uniformity
- All interactive elements must have a minimum border-radius of `8px`.
- CTA buttons should maximize the `12px` - `16px` radius for a premium, accessible feel.

### Visual Depth
- Use shadows sparingly but consistently to indicate hierarchy. Modals must have a stronger shadow than standard page cards.
