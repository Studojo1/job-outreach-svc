# Component Library

## 1. Buttons

### Primary Button
- **Visual Description**: Solid background using the module's accent color (Purple, Green, or Orange). White text.
- **Expected Usage**: Main call-to-actions, Submit buttons.
- **Spacing Rules**: `padding: 12px 24px`.
- **Border Radius**: `12px`.
- **Hover States**: Slight deepening of color or `0.9` opacity.

### Outline Button
- **Visual Description**: Transparent background with a `2px` border in the module color. Text matches border color.
- **Expected Usage**: Secondary actions, Cancel buttons.
- **Spacing Rules**: `padding: 10px 22px` (offset for border).
- **Border Radius**: `12px`.

---

## 2. Cards

### Content Card / Dojo Card
- **Visual Description**: White background (`#FFFFFF`), `1px` border (`#E5E7EB`), soft shadow.
- **Spacing Rules**: `padding: 24px` (`L`).
- **Border Radius**: `16px`.
- **Usage**: Grouping logically related sections (e.g., Experience, Profile summary, Resource links).

---

## 3. Form Inputs

### Text Input
- **Visual Description**: Light gray border, white background.
- **Spacing Rules**: `padding: 12px 16px`.
- **Labeling**: Bold labels directly above the input with `8px` (`S`) bottom margin.
- **Border Radius**: `8px`.

---

## 4. Status & Progress

### Status Badge (Capsule)
- **Visual Description**: High contrast background (e.g., Green for "DONE"), white text.
- **Shape**: Fully rounded (Pill shape).
- **Text**: `12px` Uppercase Bold.

### Progress Steps
- **Visual Description**: Circular indicators containing a number (1, 2, 3).
- **State**: Solid circle for completed/active; outlined for pending.
- **Connection**: `2px` lines connecting the circles.

---

## 5. Specialized Components

### Flashcard
- **Visual Description**: Centered card with large bold text.
- **Interactive State**: Flips horizontally on hover to reveal logic/reasoning.
- **Usage**: Explaining recruitment decisions or career matches.
